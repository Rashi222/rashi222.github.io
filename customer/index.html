Onboarding

Sign In

  • As an admin, will be able to sign into the portal using email address and password.
  • By default, Customer dashboard will be selected.
  • To show the field is active and cursor is placed on the field, placeholder would be in blue color, otherwise in grey color.
  • For Customer, Select the customer dashboard
  • Enter the valid email and password and click on sign in, you'll be logged into the portal
  • If the mail entered is of incorrect format or is left blank, will throw a validation message "Email is required and must be of valid format"
  • If the password field is left blank and clicked on sign, will throw a validation error message as "Password is required"
  • If wrong email or password is entered, will throw the notification as "Invalid email or password. Also, ensure the correct dashboard is selected"

Forgot Password

  • As you click on the forgot password button, will redirect you the screen to enter the email address in order to receive the reset password email.
  • To show the field is active and cursor is placed on the field, placeholder would be in blue color, otherwise in grey color.
  • Send rest link button would be disable until a valid email address is entered.
  • If the email entered exists in the database, the send rest link button would get activated. With a success notification of "Reset Password link sent to your email", it would redirect us to sign in page.
  • When you enter the wrong email address or leave the field blank, will show a validation error as "Email not found. Please enter a valid email."

Reset Password

  • Below is the email template with the link to reset your password From: QHL Support Subject: Reset Password

    Hi [user's first name],

Please use the link below to reset the password for your account.

Click here to reset your password
*Link access will expire in 48 hours.

Thanks
Quick Hands Logistics"

  • The link is valid for 48 hours

  • Password should follow the format of minimum 8 characters that includes 1 Uppercase, 1 Lowercase, 1 Special Character, 1 Number, else will throw a validation message
  • Reset button will be disable until the information is entered
  • Will throw a validation error message if the passwords do not match
  • If click on cancel button, will redirect you back to the sign in page
  • To show the field is active and cursor is placed on the field, placeholder would be in blue color, otherwise in grey color.
  • As you click on reset button, will throw a success notification "Password has been updated" and redirect you to the sign in page.
  • Below is the template user receives when they set their password

User Profile

As the Customer clicks on the profile icon on the top, detail information related to the logged in user would appear such as user name and password which can be further viewed and updated.

QHL logo will be visible on the top and then their information will be divided into separate sections in the following manner. 

1. Profile
2. Change Password
3. Logout

Profile

Can view the following information

  1. Contact's Full Name
  2. Profile image
  3. Associated Email id

Be able to edit the profile information by clicking on the edit button, a pop-up appears with the basic details:

  1. Profile Image
    a. A default profile image appears for the first time.
    b. Able to upload the profile image by clicking on the red upload button. 
        Clicking on upload, opens the folder from where you can select the desired image.
        As you have selected the image, the red "change" button activates which lets you change the image (can reselect the same image too)
    c. After a profile image is uploaded, can also remove the picture by clicking on the grey delete icon.
    d. As we hover over the deletion icon, it shows the help text " Remove profile photo and return to the default icon"
    e. Adding a profile image is not a mandatory field to be filled.
  1. Information such as First, middle, last name & phone no. are automatically pre-filled.
    a. The country code for a phone number is fixed as +1 and should be 10 digits.
    b. The minimum and maximum character limits for entering the name would be 1 and 20. 
    c. Middle name is not a mandatory field to be filled.

All the updated information including image, name, and phone no. will be updated only when we click on the update button.

A success notification "Contact updated successfully" will appear after successfully updating and reverts us back to the landing page of the section.

Change Password

As we click on the edit icon, a pop-up appears with the following information 

1. Enter the current password
2. Enter new password - retype the new password
    a. Can type the same password as the current too.
3. The user is prompted with the validations if:
    a. The current password is wrong as "Invalid Password"
    b. If the passwords donâ€™t match the newly typed password fields as "Password does not match"
    c. If the entered password format is wrong as "At least 8 characters that include 1 uppercase, 1 lowercase, 1 special character, 1 number"
4. Update button activates as we type all the information correctly.
5. Success notification as "Password has been changed" appears
6. User is not logged out of the portal.
7. User can also avail the benefit of the forgot password functionality by clicking on the button
    a. As we hove rover the button, the text changes to blue
    b. Redirects you to the "Forgot Password" screen

Logout

As we click on the button, it logs you out and redirects to the login screen with the same dashboard selected.

Settings
